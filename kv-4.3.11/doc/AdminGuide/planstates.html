Plan States

Plans can be in these states:

Name Description
APPROVED The plan has been created, but is not yet running.
CANCELED An INTERRUPTED or ERROR plan has been terminated using the CLI. To cancel a plan using the CLI, use the cancel command.
ERROR A RUNNING plan has encountered a problem, and has ended without successfully completing.
INTERRUPTED A RUNNING plan has been manually interrupted, using the interrupt command in the CLI.
INTERRUPT REQUESTED A plan has been manually interrupted, but is still processing the interrupt request. A plan may have to cleanup or reverse steps taken during plan execution to be sure that the store remains in a consistent state.
RUNNING The plan is currently executing.
SUCCEEDED The plan has completed successfully.

Plans in INTERRUPTED, INTERRUPT REQUESTED or ERROR state can be retried using the plan execute command. Retrying may be an appropriate approach when the underlying problem was transient or has been rectified. Plans that are retried simply re-execute the same steps. Each step is idempotent, and can be safely repeated.

Note that Storage Nodes and Replication Nodes may encounter errors which are displayed in an error dialog before the plan has processed the information. The plan eventually sees the error and transitions to an ERROR state.
